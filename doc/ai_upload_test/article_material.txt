个人网站文章素材：从想法到上线的记录

主题：
记录一个个人网站从需求文档、技术选型、页面设计、后台管理到 AI 助手接入的完整过程。

背景：
这个网站叫“B-612星球”，目标不是做一个普通的作品展示页，而是做一个可以持续记录文章、作品、评论、联系信息和成长轨迹的数字空间。

核心内容：
1. 一开始先整理需求文档，明确网站包含关于、文章、作品、联系、后台管理等页面。
2. 技术方案从 React + Vite 逐步调整为 Astro + React Islands，原因是希望页面结构更清晰，静态内容更适合展示，动态交互只在必要组件中使用。
3. 后端使用 NestJS，数据库使用 SQLite，ORM 使用 Prisma。
4. 后台管理包括文章管理、作品管理、评论管理、联系信息管理、网站设置和 AI 助手。
5. AI 助手使用 DeepSeek，分为管理员和访客两种角色。管理员可以生成内容、分析内容、上传文件解析；访客只能基于公开内容提问。

可写成文章的角度：
- 为什么个人网站也需要需求文档
- 为什么从 React + Vite 调整到 Astro + React Islands
- 如何一步步把个人网站从静态页面做成带后台和 AI 的系统
- AI 助手在个人网站中的边界：辅助创作，但不替代管理员判断

文章风格：
自然、真诚、有一点项目复盘感，不要太像教程，也不要太商业化。

建议标签：
个人网站、Astro、React、NestJS、SQLite、AI 助手、项目复盘
