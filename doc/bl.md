# 个人网站需求文档

> 文档状态：持续迭代  
> 最后更新：2026-07-20  
> 项目名称：个人星球

## 1. 项目概述

### 1.1 项目背景

建设一个展示个人经历、作品、文章与思考的独立个人网站，作为长期维护的数字名片与内容空间。

### 1.2 项目目标

- 清晰展示个人介绍、文章、作品与联系方式。
- 支持访客浏览内容、评论互动和明暗主题切换。
- 支持管理员在网站内新增、编辑文章和作品。
- 保持良好的移动端体验、视觉表现与后续可维护性。

### 1.3 目标用户

- 普通访客：浏览个人介绍、文章和作品。
- 招聘方、合作方：快速了解能力、经历与项目成果。
- 管理员：维护网站内容、回复或管理互动内容。

## 2. 网站定位与视觉风格

### 2.1 品牌信息

| 项目 | 内容 |
| --- | --- |
| 网站名称 | 个人星球 |
| 品牌描述 | 把生活的灵感画进数字宇宙 |
| 明亮主题 | 暖米色、橙红、自然绿；手绘、温暖、轻盈 |
| 暗色主题 | 深空黑、深蓝、霓虹青、紫与粉；科幻、HUD、宇宙感 |

### 2.2 导航规范

- 网站采用独立页面路由，不使用单页锚点替代页面跳转。
- 导航栏固定在页面顶部，随当前明暗主题改变底色。
- 导航显示：首页、关于、文章、作品、联系；管理员登录后额外显示“后台”。
- 当前页面导航项应有高亮和动态过渡效果。
- 导航栏提供明暗主题切换按钮。
- 管理员退出按钮仅在管理员登录后显示。

## 4. 页面需求

### 4.1 首页

- 首屏展示欢迎语、个人简介、开始浏览按钮和右侧星球插画。
- 首页右侧星球插画是管理员登录入口。
- 星球插画应保留悬浮、点击和键盘聚焦反馈。
- 首页下方展示站点数据、下一步导览卡片和页脚。

### 4.2 关于页面

- 展示个人简介、技能栈、经历、兴趣与个人理念。
- 支持时间轴、技能标签、信息卡片与滚动加载动画。

### 4.3 文章页面

- 文章列表展示标题、摘要、标签、日期和阅读时长。
- 每篇文章可进入独立详情页。
- 文章详情展示标题、摘要、正文、返回列表入口、评论区和管理员修改入口。
- 管理员保存内容后，详情页刷新时应展示后端最新标题、摘要和正文。

### 4.4 作品页面

- 作品列表展示项目名称、简介、封面风格和技术栈。
- 每个作品可进入独立详情页。
- 作品详情展示项目目标、技术栈、状态、设计过程、评论区和管理员修改入口。
- 管理员保存内容后，详情页刷新时应展示后端最新标题、摘要和正文。

### 4.5 联系页面

- 展示邮箱、GitHub 和其他社交联系方式。
- 提供联系表单和订阅入口。
- 联系页面不需要长页面滚动效果。

### 4.6 页脚与返回顶部

- 每个页面均展示页脚，页脚适配明暗主题。
- 页脚进入视口时有一次性出现动画。
- 页面右下角提供固定的返回顶部按钮，带悬浮与点击动画。

## 5. 功能需求

| 模块 | 基础需求 | 后续增强 |
| --- | --- | --- |
| 内容浏览 | 文章、作品列表与详情页 | 分类、标签、全文搜索 |
| 主题 | 明暗主题切换并记住偏好 | 跟随系统主题 |
| 互动 | 评论、点赞、评论排序 | 回复、审核、通知 |
| 管理 | 登录、退出、新增、编辑内容 | 草稿、定时发布、删除、版本记录 |
| 导航 | 独立路由、当前页高亮、返回顶部 | 面包屑、站内快捷搜索 |
| 多端适配 | 手机、平板、电脑可用 | 更完善的无障碍支持 |

### 5.1 评论与点赞

- 每篇文章详情页和每个作品详情页底部均展示独立评论区。
- 每条评论显示默认头像、昵称、评论内容、发布时间和点赞数。
- 用户可以填写昵称与评论内容并发布评论。
- 评论支持“最新”和“点赞最多”排序，默认按最新排序。
- 同一浏览器会话中，每条评论只能点赞一次。
- 评论表单、卡片、点赞按钮应适配明亮和暗色主题，并有进入与悬浮动画。
- 后端接口：
  - GET /api/comments：读取评论列表。
  - POST /api/comments：发布评论。
  - POST /api/comments/:id/like：为评论点赞。

### 5.2 管理员登录与退出

- 首页右侧星球插画为管理员登录入口；导航栏“个人星球”文字只返回首页。
- 点击星球后弹出登录窗口。
- 当前开发演示账号为 admin，密码为 123456。
- 登录成功后，将管理员会话令牌保存到浏览器本地，并在导航栏显示“后台”入口。
- 管理员可通过导航栏退出按钮结束会话。
- 无有效管理员会话时，不显示详情页修改按钮，且后端拒绝管理接口请求。
- 后端接口：
  - POST /api/auth/login：登录。
  - POST /api/auth/logout：退出登录。

### 5.3 管理后台（当前内容管理）

- 管理员登录后可访问 /admin。
- 页面采用左右双栏布局：左侧为内容选择、新增操作和编辑表单；右侧为与实际详情页结构一致的实时预览。
- 可编辑字段：标题、摘要、详情正文。
- 输入内容时，右侧预览应立即同步变化，无需保存。
- 管理员可选择已有文章或作品继续编辑。
- 提供“新增文章”和“新增作品”按钮；新建后可保存并继续编辑。
- 后端接口：
  - GET /api/admin/content：读取管理员可编辑内容。
  - POST /api/admin/content：新增文章或作品。
  - PATCH /api/admin/content/:type/:id：更新文章或作品。

### 5.4 详情页管理员修改入口

- 管理员登录时，文章和作品详情页显示“修改内容”按钮。
- 点击按钮跳转到 /admin，并自动选中当前文章或作品。
- 非管理员访客不可见该按钮。

## 6. 交互动效需求

- 页面模块进入视口时从下向上渐进出现。
- 文章与作品详情页的进入动画仅播放一次，离开后再次进入不重复播放。
- 卡片具备轻微的立体悬浮、鼠标移动和阴影变化效果。
- 文字悬停时使用可变字体或字形变化效果，不应影响可读性。
- 明暗主题切换应平滑过渡。
- 首页星球插画悬停时应提示其可作为管理员入口。

## 7. 技术方案

### 7.1 当前技术栈

- 前端：Astro + React Islands + TypeScript。
- 样式：原生 CSS，使用 CSS 变量管理明暗主题。
- 动效：CSS 动画与 Motion React。
- 后端：NestJS + TypeScript。
- 开发环境：Vite / Astro Dev Server。

### 7.2 数据与接口

- 当前评论、管理员会话和内容数据暂存于 NestJS 服务内存。
- 前端通过 HTTP API 读取评论、管理员内容和详情页更新内容。
- 生产环境应使用数据库保存内容、评论、账号与会话。

## 8. 非功能需求

### 8.1 性能

- 首屏优先加载静态内容和必要资源。
- 动效应避免造成明显卡顿。
- 移动端应控制动画数量与资源体积。

### 8.2 兼容性

- 支持现代 Chromium、Firefox、Safari 浏览器。
- 支持常见手机、平板和桌面端宽度。
- 对不支持复杂动效的环境保留可阅读的静态内容。

### 8.3 安全与上线要求

- 开发演示账号和密码不得直接用于公开生产环境。
- 上线前应将管理员账号、密码和密钥迁移至环境变量。
- 密码必须使用哈希存储，不得保存明文。
- 管理员令牌应具备过期机制；管理接口应增加鉴权、限流和日志。
- 评论应具备内容审核、反垃圾、防重复点赞和异常处理能力。
- 建议部署于支持国内访问的云服务器或静态托管与 API 服务组合。

## 9. 当前阶段限制

- 管理员账号、会话、评论和内容当前存放在内存，后端重启后会恢复初始数据。
- 文章和作品公开列表目前由 Astro 静态生成；新增内容尚不会自动进入公开列表或生成独立静态详情链接。
- 下一阶段应将公开列表和详情路由改为可读取数据库或动态 API 数据的实现。

## 10. 验收标准

- 普通访客可浏览首页、关于、文章、作品、联系和详情页。
- 明亮与暗色主题下文字、输入框、卡片和评论区均保持清晰可读。
- 用户可发布评论、点赞，并切换两种排序方式。
- 点击首页右侧星球可打开管理员登录窗口。
- 管理员登录后可看到“后台”导航项、退出按钮和详情页“修改内容”按钮。
- 管理员可在后台新增或编辑文章、作品，并在右侧实时预览。
- 管理员保存后，已有详情页刷新可显示更新后的标题、摘要和正文。
- 管理员退出后，管理入口和详情页修改按钮消失。
## 11. 前后端分离与项目目录规范

项目采用“同仓库、独立应用”的前后端分离模式：前端只处理页面、交互和接口调用；后端处理鉴权、数据校验、业务规则与数据库访问；双方通过 HTTP REST API 传递 JSON 数据。

```text
Company_internship_person_web/

├─ code_src/
│  ├─ package.json                   # Workspace 根脚本
│  ├─ .gitignore
│  ├─ client/                        # 前端：Astro + React Islands
│  │  ├─ public/                     # 图片、图标等静态资源
│  │  ├─ src/
│  │  │  ├─ pages/                   # 页面路由
│  │  │  ├─ components/              # React 交互组件
│  │  │  ├─ layouts/                 # 公共布局
│  │  │  ├─ lib/                     # API 请求与通用工具
│  │  │  ├─ types/                   # TypeScript 类型
│  │  │  ├─ styles/                  # 全局和主题样式
│  │  │  └─ data/                    # 静态展示或离线兜底数据
│  │  └─ package.json
│  └─ server/                        # 后端：NestJS + Prisma
│     ├─ prisma/                     # Prisma Schema 与迁移
│     ├─ data/                       # SQLite 运行数据库，不提交 Git
│     ├─ src/
│     │  ├─ main.ts                  # 服务启动、CORS、全局管道
│     │  ├─ prisma/                  # PrismaService 与连接模块
│     │  └─ modules/                 # 后续按业务模块拆分
│     ├─ .env                        # 本地配置，不提交 Git
│     └─ package.json
└─ README.md                         # 项目总说明
```

### 11.1 数据流与边界

```text
浏览器 / Astro + React
        ↓ fetch / JSON
NestJS REST API（鉴权、校验、业务规则）
        ↓ Prisma ORM
SQLite（内容、评论、会话、订阅等数据）
```

- 客户端接口根地址统一放在 `client/src/lib/api.ts`，生产环境使用 `PUBLIC_API_BASE` 配置后端地址。
- 正式内容、评论、管理员会话、联系表单和订阅数据以数据库接口返回结果为准；`client/src/data/` 仅用于静态展示或离线兜底。
- `dist/`、`.astro/`、`node_modules/`、`.env` 和 `server/data/*.db` 属于生成文件、依赖或敏感运行数据，不提交 Git。
- 后端业务继续增长时，应把现有接口逐步迁移为 `auth`、`contents`、`comments`、`admin` 等独立模块，避免把业务逻辑继续堆积在 `main.ts`。
### 11.2 内容数据读取规范

- 文章列表页面在每次请求时调用后端 `GET /api/articles`；作品列表页面调用 `GET /api/projects`。
- 文章详情页调用 `GET /api/content/article/:id`；作品详情页调用 `GET /api/content/project/:slug`。作品接口同时支持以数据库 ID 或公开 slug 查询。
- Astro 前端采用 Node 服务端渲染（SSR），因此管理员发布或修改内容后无需重新构建前端，刷新对应页面即可读取 SQLite 中的最新数据。
- `client/src/data/content.ts` 不再作为公开文章和作品页面的数据源，仅可保留为开发期参考或离线兜底；后续可在确认不再需要后删除。
- 生产环境需分别运行后端 NestJS 服务与前端 Astro 服务；前端通过 `PUBLIC_API_BASE` 指向后端 API 地址。
## 12. 管理后台（已实现）

### 12.1 名称、入口与路由

- 管理员登录后，前台导航栏显示“后台”入口；保留 `/admin` 作为后台首页和兼容已有详情页编辑链接的入口。
- `/admin` 首期展示后台概览与快捷入口；原有文章、作品编辑能力逐步迁移到独立分页面。
- 后台路由仅允许拥有有效管理员令牌的用户访问；未登录或令牌过期时跳回首页或登录提示。

```text
/admin                    后台概览
/admin/articles           文章管理
/admin/projects           作品管理
/admin/comments           评论管理
/admin/messages           联系信息管理
/admin/settings           网站设置
```

### 12.2 后台通用布局

- 桌面端采用左侧固定侧边栏和右侧内容区域；移动端侧边栏折叠为抽屉菜单。
- 侧边栏固定显示：概览、文章、作品、评论、联系信息、网站设置，以及退出登录。
- 每个分页面显示面包屑、页面标题、数据总数、查询/筛选区、列表区和分页区。
- 危险操作（删除、隐藏评论、清空信息）必须二次确认；所有管理员操作写入 `audit_logs`。

### 12.3 文章管理 `/admin/articles`

- 列表展示标题、状态、标签、发布时间、更新时间、评论数与操作入口。
- 支持新建、编辑、预览、发布/撤回、归档、删除、按状态或关键词筛选。
- 编辑页保留现有实时预览能力；保存后返回文章 ID 与公开详情链接。
- 接口规划：`GET/POST /api/admin/articles`、`GET/PATCH/DELETE /api/admin/articles/:id`。

### 12.4 作品管理 `/admin/projects`

- 列表展示作品名称、slug、技术栈、状态、更新时间与公开链接。
- 支持新建、编辑、预览、发布/撤回、归档、删除和 slug 唯一性校验。
- 接口规划：`GET/POST /api/admin/projects`、`GET/PATCH/DELETE /api/admin/projects/:id`。

### 12.5 评论管理 `/admin/comments`

- 列表展示评论者、关联文章/作品、评论内容、点赞数、创建时间与当前状态。
- 已支持按关联内容类型（文章/作品）与状态筛选，并可按最新时间或点赞数量排序；支持显示、隐藏、标记垃圾和删除。
- 默认不允许管理员直接伪造访客点赞；删除和状态变更必须写入审计日志。
- 接口规划：`GET /api/admin/comments`、`PATCH /api/admin/comments/:id`、`DELETE /api/admin/comments/:id`。

### 12.6 联系信息管理 `/admin/messages`

- 读取 `contact_messages` 表，展示称呼、邮箱、内容、接收时间和处理状态。
- 支持未读、已读、已回复、归档状态切换；支持按邮箱/关键词/时间筛选。
- 联系内容仅管理员可见，不在前台公开；导出数据前需二次确认。
- 接口规划：`GET /api/admin/messages`、`PATCH /api/admin/messages/:id`、`DELETE /api/admin/messages/:id`。

### 12.7 网站设置 `/admin/settings`

- 基于 `site_settings` 表管理站点名称、首页介绍、联系邮箱、GitHub 地址、社交链接、SEO 标题/描述和页脚版权信息。
- 设置按键值对保存；前端读取公开设置时不得暴露管理员密钥、会话或内部配置。
- 提交设置后提供预览和“恢复默认值”入口；关键设置变更写入审计日志。
- 接口规划：`GET /api/admin/settings`、`PATCH /api/admin/settings`，以及只读公开接口 `GET /api/settings/public`。

### 12.8 当前实现状态

- 已实现 `/admin` 概览页，统计文章、作品、可见评论和未读联系信息。
- 已实现文章与作品的独立编辑页，支持新增、修改以及详情预览。
- 已实现评论显示/隐藏、删除，联系信息状态更新/删除，以及网站设置的键值编辑。
- 所有后台请求均需管理员 Bearer 登录令牌；内容、评论、联系信息和设置的管理操作会写入 `audit_logs`。
- 后续可补充分页、关键词筛选、批量处理、内容恢复、公开站点设置读取和多管理员权限。
### 12.9 内容字段与文章/作品管理（已实现）

- `/admin/articles` 与 `/admin/projects` 分别管理文章和作品；各自只显示和操作同类型内容。
- 两类内容均支持新增、编辑与删除。删除会级联删除关联的 `content_tags` 和评论等关系数据，并写入审计日志。
- 编辑字段包括标题、摘要、正文、`cover_url`（封面图地址）、`status`（`draft`、`published`、`archived`）和标签。
- 作品额外支持编辑 `stack` 技术栈；文章保存时该字段固定为空。
- 标签使用逗号输入，保存时去除空白和重复项，写入 `content_tags`；更新内容时会同步替换该内容的全部标签。
- 仅 `published` 内容会由公开 API 返回。后台中已发布内容可通过“打开真实页面”进入文章 `/articles/:id` 或作品 `/projects/:slug` 的实际展示页面；草稿和归档内容不提供公开入口。
