## 4. 页面需求

### 4.1 首页

- 首屏展示：用动态背景，先展示欢迎界面
- 内容重点：让访客被吸引

### 4.2 关于我

- 个人简介、技能栈、教育或工作经历。
- 可展示头像、兴趣爱好、个人理念。
- 可选：时间轴形式展示经历。

### 4.3 文章 / 博客

- 文章列表支持按时间、分类、标签浏览。
- 文章详情支持目录、代码高亮、图片、上一篇 / 下一篇。
- 可选功能：全文搜索、阅读时长、点赞或评论。

### 4.4 作品集

- 以动画卡片形式展示项目名称、简介、封面与技术栈。
- 项目详情包含：背景、职责、成果、截图、在线地址与代码地址。

### 4.5 简历（可选）

- 在线展示简历内容。
- 可选：提供 PDF 下载。

### 4.6 联系我

- 展示邮箱、社交账号、GitHub 等联系方式。
- 可选：联系表单与留言功能。

## 6. 非功能需求

### 6.1 性能

- 页面应在常规网络环境下快速打开。
- 图片应压缩并支持懒加载。

### 6.2 兼容性

- 优先支持现代浏览器：Chrome、Edge、Safari、Firefox。
- 支持手机端与桌面端的响应式布局。

### 6.3 SEO 与分享

- 每个页面配置独立的标题、描述与关键词。
- 文章页支持分享卡片（Open Graph）信息。
- 自动生成站点地图与 robots.txt。

### 6.4 安全与隐私

- 全站启用 HTTPS。
- 联系表单需具备基础防垃圾提交能力。
- 明确说明收集的访客数据（如启用统计或表单）。

## 8. 技术与部署方案

### 8.1 上线前提：面向中国大陆访客

- 若网站部署在中国大陆服务器并绑定域名，需先完成 ICP 备案；备案信息应与实际主办者、域名和接入服务商保持一致。
- 若使用中国大陆服务器，通常还应在网站开通后按要求完成公安联网备案；上线前以当地管局和云服务商的最新规则为准。
- 域名需完成实名认证；页面底部展示备案号并链接至工信部备案查询站点。
- 避免将网站关键资源依赖于在中国大陆可能不稳定或不可访问的第三方服务；图片、字体、代码库和统计服务优先选择国内可用服务或自托管。

参考： [腾讯云 ICP 备案指引](https://cloud.tencent.com/document/product/243/39038)、[接入备案说明](https://cloud.tencent.com/document/product/243/37403)。

### 8.2 方案一：静态博客 + 国内对象存储 / CDN（推荐作为首发版本）

| 项目 | 选择建议 |
| --- | --- |
| 前端框架 | Astro（优先）或 VuePress / VitePress |
| 内容管理 | Markdown 文件随代码维护，可通过 Git 提交发布文章 |
| 部署 | 阿里云 OSS + CDN，或腾讯云 COS + CDN；也可使用国内云厂商的静态网站托管 |
| 动态功能 | 先使用第三方评论 / 表单服务，后续按需迁移到自建接口 |
| 适合阶段 | 内容较少、以博客和作品展示为主、希望成本与维护量最低 |

**特点：** 页面预先生成，访问速度快、安全面小、部署成本低；适合先完成首页、文章、作品集和联系页面。局限是编辑文章需要使用 Git / Markdown，评论、搜索和订阅等互动功能需要额外服务。

### 8.3 方案二：Nuxt 3 全栈网站 + 国内云服务器（推荐作为长期主站）

| 项目 | 选择建议 |
| --- | --- |
| 前端与服务端 | Nuxt 3 + TypeScript；前端页面与 API 同一项目维护 |
| 内容管理 | Nuxt Content（Markdown）起步；后续可接入 Directus / Strapi 等 Headless CMS |
| 数据库 | MySQL 或 PostgreSQL，使用云数据库或与应用同机部署 |
| 部署 | 腾讯云轻量应用服务器 / CVM，Nginx + Docker；搭配国内 CDN 与对象存储 |
| 适合阶段 | 需要后台管理、评论、搜索、订阅、留言等持续互动功能 |

**特点：** 能在同一技术栈中实现前台、管理接口和动态功能，扩展性强；适合把个人博客长期运营为有内容管理能力的主站。代价是需要维护服务器、数据库、备份、监控与安全更新，首次配置和运维成本高于静态方案。

### 8.4 方案三：前后端分离 + 托管 CMS + Serverless（适合内容运营）

| 项目 | 选择建议 |
| --- | --- |
| 前端框架 | Next.js 或 Nuxt 3，优先使用静态生成与按需刷新 |
| 内容管理 | Directus / Strapi 等 Headless CMS，供浏览器后台编辑文章和作品 |
| 后端能力 | 腾讯云云函数 / 云托管，或阿里云函数计算；对象存储保存图片与附件 |
| 数据与搜索 | 云数据库 + 国内可用的搜索服务或自建 Meilisearch |
| 适合阶段 | 希望非技术方式发布内容，且后期会增加表单、订阅、互动和多端内容调用 |

**特点：** 内容编辑体验最好，前端与内容后台职责清晰，动态能力可按调用量扩展；但涉及的云产品较多，权限、成本和备份策略需要统一规划。使用大陆地区 Serverless 绑定自定义域名同样需要完成 ICP 备案。

### 8.5 方案选择建议

- **当前建议：** 先采用方案一，用 Astro + Markdown + 国内对象存储 / CDN 快速完成 MVP；网站结构稳定后，再按“需要后台管理和互动功能”的优先级升级到方案二。
- **需要浏览器后台写作：** 直接选择方案三。
- **需要强互动、会员或长期自定义功能：** 选择方案二。

### 8.6 方案四：React + Vite 前后端分离（动态视觉与互动优先）

| 项目 | 选择建议 |
| --- | --- |
| 前端框架 | React + Vite + TypeScript |
| 动效与交互 | GSAP 或 Framer Motion 制作页面动效；Lenis 优化平滑滚动体验 |
| 后端服务 | NestJS（优先）或 Express，提供内容、评论、订阅、搜索和联系表单 API |
| 内容管理 | 自建管理后台；文章可先使用 Markdown，后续迁移到 Headless CMS |
| 数据与资源 | MySQL / PostgreSQL + 国内对象存储 + 国内 CDN |
| 部署 | 国内云服务器，Nginx 分发前端静态文件并反向代理后端 API |
| 适合阶段 | 重视手绘 / 油画风动效、复杂前端互动，且团队熟悉 React 的长期网站 |

**特点：** React 生态适合制作复杂的视觉交互，Vite 提供快速的本地开发体验；通过独立 Node.js 后端可实现后台管理、评论、搜索、订阅与表单。默认采用客户端渲染，文章 SEO 与首屏性能需要通过预渲染或 SSR 额外优化；若后续 SEO 成为核心目标，可评估迁移至 Next.js。

**部署要点：** 前端构建产物可部署至对象存储 / CDN，后端 API 与数据库部署在国内云服务器；绑定国内服务器时需按要求完成域名实名认证与备案。

## 9. 优先级与迭代计划

### 第一阶段：最小可用版本（MVP）

- [ ] 分类、标签、搜索与明暗主题。
- [ ] 项目详情、在线简历与内容统计。

### 第三阶段：互动与增长

- [ ] 评论、订阅、留言或其他社区功能。
